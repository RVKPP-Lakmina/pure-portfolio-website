Building My First Full Stack Application

The Journey of Developing a Complete Web Application

Building My First Full Stack Application

By Pramesh Lakmina | October 10, 2024

As a developer, nothing quite beats the excitement and satisfaction of building your first full stack application from scratch. It's a rite of passage that every software engineer experiences, and for me, it marked a turning point in my career. In this blog post, I’m going to walk you through the process of creating my first full stack web application using React, Node.js, Express, and MongoDB.

The application I built was a simple to-do list app that allowed users to create, edit, and delete tasks. Though the project seems simple at first glance, it allowed me to learn a vast array of skills, including front-end development, back-end programming, database management, and deployment. By the end of this post, you'll have a clear picture of what it's like to develop a full stack application and the challenges I faced along the way.

Project Overview

The idea for the project came from my desire to build something that was not only functional but also practical for day-to-day use. I chose a to-do list app because it’s an application I use regularly, and I wanted to make one that worked exactly the way I needed it to. It involved creating a front-end where users could interact with the app, a back-end API to manage the data, and a database to store the tasks.

Technologies I used for the project include:

  • React.js for the front-end user interface
  • Node.js with Express for the back-end API
  • MongoDB as the database
  • GitHub for version control
  • Heroku for deploying the application

The Front-End: React

Building the front-end was my first exposure to React, and I immediately fell in love with its component-based architecture. It was so much more intuitive than writing raw HTML, CSS, and vanilla JavaScript. In React, each part of my application became a reusable component, whether it was the input form for adding tasks, the list displaying all tasks, or the button for deleting them.

The learning curve for React was initially steep, but once I grasped the concept of components, JSX (a syntax extension for JavaScript that allows HTML-like code), and the state, things started to make sense. I used hooks like useState and useEffect to manage the state and lifecycle of my components.

Challenges I faced with React:

  • Understanding the proper way to pass data between parent and child components
  • Managing state efficiently without causing unnecessary re-renders
  • Learning how to handle side effects, like fetching data from the back-end API, with useEffect

The Back-End: Node.js & Express

After building the front-end, it was time to move to the back-end. This was where I set up an API that the React app could communicate with. I chose Node.js for the back-end because of its fast, event-driven, non-blocking I/O model. Express, a popular Node.js framework, made it easier to set up routes for handling HTTP requests.

I created a simple RESTful API that allowed the following operations:

  • POST: Create a new task
  • GET: Retrieve all tasks
  • PUT: Update an existing task
  • DELETE: Remove a task

Challenges I faced with Node.js & Express:

  • Setting up proper error handling in Express routes
  • Managing asynchronous requests and ensuring data consistency
  • Securing the API with proper authentication (which I added later)

Working with MongoDB

For data storage, I chose MongoDB, a NoSQL database. One of the reasons I selected MongoDB was because of its flexibility with unstructured data and the ease of integration with Node.js through the mongoose package. In my app, each task was represented as a document in a MongoDB collection. This collection stored information like the task description, completion status, and timestamps.

Setting up MongoDB required me to learn about:

  • Creating schemas with Mongoose
  • Connecting the MongoDB database to my Express app
  • Performing CRUD operations (Create, Read, Update, Delete)

Challenges I faced with MongoDB:

  • Defining the schema structure correctly to ensure flexibility
  • Handling database errors gracefully
  • Ensuring that the data was always in sync with the front-end

Deployment to Heroku

Once the app was functioning locally, I had to deploy it to make it accessible to others. I chose Heroku for deployment because of its simplicity and seamless integration with GitHub. With Heroku, I could continuously deploy my app every time I pushed new code to the repository. The process involved setting up a Procfile and configuring environment variables for MongoDB and other sensitive data.

Deploying my app exposed me to concepts like:

  • Environment variables and their role in securing sensitive information
  • Configuring Heroku dynos to ensure the app ran efficiently
  • Managing scaling concerns as more users began accessing the application

Key Learnings

Building this full stack application was an incredible learning experience. I learned how to structure a web app from the front-end to the back-end, how to handle asynchronous programming, and how to deploy an app in a production environment. While I faced many challenges, the satisfaction of seeing everything come together was worth every late night spent debugging code.

Future Improvements

Looking back, there are several areas where I can improve this project:

  • Adding authentication so that users can sign in and manage their own tasks
  • Improving the user interface by using more sophisticated CSS or a framework like Bootstrap
  • Optimizing the performance of the app as the task list grows larger
  • Integrating automated testing for both the front-end and back-end

Conclusion

In conclusion, building a full stack application is not only a great way to learn about different technologies, but it also provides a sense of accomplishment that few other projects can offer. The process of combining a front-end, a back-end, and a database into one cohesive application is immensely rewarding. If you're a developer looking to take your skills to the next level, I highly recommend starting your own full stack project.
